## Current Governance

### Project Philosophy

The project follows these principles:

* Evidence over assumptions.
* Small, testable changes.
* Stable foundation.
* Explain the "why," not just the rule.
* Modular design.
* Documentation is part of the code.

### Documentation Standard

Documentation is written for people who want to understand and troubleshoot Home Assistant systems.

Every document should be understandable without requiring technical experience or outside resources.

Technical terms are used only when they improve communication. When a technical term is introduced, it is explained in everyday language before it is used throughout the rest of the document.

The First Reader should be able to read a document without stopping to ask what something means. If the First Reader has a question, the document should be rewritten rather than expecting the reader to search somewhere else for the answer.

## Important Development Approach

The project follows these practices throughout development:

* Work on one problem at a time.
* Observe before reaching conclusions.
* Understand the cause of a problem before trying to solve it.
* Make one change at a time and verify the results before making another.

Every improvement should make both the software and the documentation better.

The goal of the Home Assistant Diagnostic Suite is not only to build useful diagnostic tools, but also to create documentation that helps people understand how those tools work and why they were designed that way.
